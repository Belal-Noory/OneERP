# OneERP Modules

Each module has a product spec in this folder. During development we follow these specs to keep UX, permissions, and audit logging consistent.

## How to use these docs

- Update the relevant module spec before implementing a feature.
- When a feature is implemented, mark it in the checklist section of the spec.
- Any API endpoint or DB change must be reflected in the spec.
